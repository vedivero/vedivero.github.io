Java Virtual Machine


자바번역기라고도 불리는 자바가상기계(JVM)는 자바 바이트 코드와 컴퓨터의 운영 시스템 간에 번역기 역할을 하여 자바로 작성된 응용 프로그램을 윈도나 매킨토시, 유닉스 등에서 사용할 수 있도록 해주는 소프트웨어를 말한다.

JVM을 알아야 하는 이유


동일한 기능의 프로그램이라도 메모리 관리에 따른 성능 차이 발생

한정된 메모리를 가장 효율적으로 사용하여 최고의 성능을 내기 위함

→ 성능에 치명적인 메모리 누수 관리를 위해 JVM의 구조를 파악


Java 프로그램 실행 과정


  1. 프로그램 실행 시, JVM은 실행에 필요한 메모리를 OS로부터 할당
  2. 자바 소스코드(.java)를 자바 컴파일러(javac)가 읽은 후 → 자바 바이트코드로 변환
  3. Class Loader가 .class파일을 JVM에 로딩
  4. 로딩된 .class파일은 Execution engine<을 통해 해석
  5. 해석된 .class파일의 코드는 Runtime Data Areas 에 배치 및 수행


JVM 구성

1. Class Loader
  • 클래스파일을 읽고, 링크를 통해 배치하는 작업을 수행하는 모듈
  • .java 파일을 컴파일하면 .class(바이트코드)가 생성
  • class파일을 Runtime data Aread 형태로 메모리에 적재하는 역할
2. Execution Engine
  • 적재된 클래들을 기계어로 번역해 실행

기계어를 번역하는 2가지 방법

1) Interpreter(인터프리터) 방식

  • 실행 엔진은 바이트코드 단위로 읽어서 실행
    단점 : 한 줄 씩 수행하여 속도가 느림.

2) JIT(Just In Time)

  • 인터프리터의 단점을 보안한 JIT 컴파일 방식

  • 어느 시점에서 바이트코드 전체를 네이티브 코드로 변경하여 작업을 수행

  • 한번 변경한 코드를 캐시에 보관하기에 수행 속도가 빠름

단점: 네이티브 코드로 번역 시 시간이 소요되므로 자주 수행되지 않을 경우 좋지 않음

3. Garbage Collector
  • Java에는 GC를 수행하는 모듈(쓰레드)이 존재
  • Heap 메모리에 생성된 객체 중 Reachabiliry를 잃은 객체를 탐색 후 제거하는 역할
4. Runtime Data Aread

1) Method Area(= Class Area = Static Area)

  • 클레스 멤버 변수, 메소드 정보, Type정보, static, final 변수 등 생성
  • 상수 풀(Static Area)은 모든 Symbolic Reference를 포함

클래스 정보를 메모리 공간에 처음 로드할 때 초기화되는 대상을 저장하기 위한 메모리 공간

2)Heap Area

  • 객체를 저장하는 가상 메모리 공간
  • new연산자를 통해 생성된 객체와 배열을 저장

Method Area에 로드 된 클래스만 객체로 생성가능

Heap Memory의 영역

  • New/Young 영역

  • Permanent Generation

  • old영역

3) Stack Area

  • 프로그램 실행 시 임시로 할당 후 메소드가 종료되면 소멸되는 특성의 데이터를 저장하기 위한 공간

매개 변수, 지역 변수, 리턴 값 및 연산이 일어나는 값이 임시로 보관될 때 사용되는 영역

4) PC Register

  • Thread가 시작될 때 생성 생성될 때마다 해당 공간으로 스레드마다 하나씩 존재

어떤 부분을 어떤 명령으로 실행해야할지에 대한 기록하는 부분

현재 수행중인 JVM 명령의 주소를 보관

5) Native Method Stack

  • 컴파일 시 생성 된 바이트코드가 아닌 실제 실행할 수 있는 기계어로 작성된 프로그램을 실행시키는 영역, JAVA가 아닌 다른 언어로 작성된 코드를 위한 공간
